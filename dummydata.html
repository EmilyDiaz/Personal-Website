Dummy Data Generator

Overview

Problem

Tables are a common component used throughout the Appeals management software and their data varies based on many factors, such as the type of user viewing the table, the queue a table is in, etc. Filling out tables with the appropriate data and unique data is time consuming, but failure to do so during handoff causes confusion for developers, resulting in more unneccessary meetings or syncs to explain discrepencies.

Challenges faced

  • Lack of familarity with the Figma API and typescript.

Timeline

August 2024-February 2025

Role

UX Designer & Developer

Solution breakdown

Create a Figma plugin that helps automate the table data process.

Dummy Data Generator allowed UX designers to quickly fill up table components in Figma with more accurate and unique data than just a typical lorem ipsum placeholder. Having unique placeholder text for different table columns helped us deliver more authentic prototypes to the developers, allowing us to illustrate different flows in which, for example, required different Veteran names or unique data for each veteran.

This plugin was created before Figma introduced its AI enhancments on the canvas, such as the replace-content feature. That feature rendered this plugin a little less useful, though the plugin still proved quicker to use than having to continuously correct the replace-content AI feature.

This was the first plugin I made. The figma documentation for its plugin API is a bit...sparse, so I did create my plugins with the help of AI, specifically Github Copilot's Claude model.
